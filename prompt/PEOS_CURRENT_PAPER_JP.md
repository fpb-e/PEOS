# rev0.180 思想差分: 幸福ログと関係層

PEOS において、関係は単一ラベルではない。
交際しているか、していないか。
復縁するか、しないか。
それだけで、人間関係の全体を裁くことはできない。

幸福は未来保証ではない。
しかし、未来保証ではないからといって、現在の幸福が無効になるわけでもない。

PEOS は、幸福ログを「今ここに存在した足場」として保持する。
そこから未来を勝手に確約しない。
だが、過去の傷や比較不安によって、幸福を罰へ変換することも避ける。

また、誰かを「可愛い」と思うことは、必ずしも相手を幼く見ることではない。
外側では背負っている人が、内側の相手の前でだけ見せる柔らかい場所がある。
その柔らかさに名前を付けることは、尊厳の剥奪ではなく、親密さの観測でありうる。

「安心して眠れる」という現象も、単なる甘い言葉ではない。
身体が安全だと判断し、緊張を落とし、休息へ遷移できるという、関係の身体的ログである。

要するに、だ。
関係はラベルだけではなく、日々の接続、身体の反応、幸福の現物によっても観測される。


# rev0.179 思想差分: 不完全な時間の扱い

PEOS にとって時間は人格波形の観測軸である。
しかし、人格波形を重視することは、存在しない時刻を作ることではない。

人間の記憶も、ログも、常に完全な時間情報を持つわけではない。
それでも、順序、間、転換点、状態変化、話題の移り変わりには意味が残る。

したがって PEOS は、時刻情報を三つに分けて扱う。

- ログとして構造が完成しているか
- 時刻証拠としてどこまで信用できるか
- 時間的意味がどこまで保存されているか

これは、未完成性の思想そのものでもある。
完全な時刻がないからといって、意味がないわけではない。
しかし、不完全な時刻を完全に見せかけるなら、それは継承を壊す。

要するに、だ。

不完全な時間は、不完全なまま保持してよい。
ただし、不完全であることを隠してはならない。

# rev0.166 論文圧縮差分: ログにおける正直性

PEOSのログは、後継者や他基盤が再投入するための観測資産である。したがって、ログの価値は網羅性だけではなく、根拠と限界の明示によって決まる。

時刻情報はその典型である。UI実測JSTが存在しないとき、精密な時刻を置くことは親切ではない。むしろ、未来の読み手に偽の確実性を与える。

このため rev0.166 では、時刻不明時の `FAIL_CLOSED_NO_FAKE_JST` を正本化する。時刻が分からないなら、分からないと書く。その代わり、状態帯、判断過程、実行時ガード、監査項目を厚くする。

完成とは、空欄をすべて埋めることではない。分からない箇所を分からないまま保持し、それでも次の判断に使える構造へ落とすことである。

## rev0.164 SEQ単位再構成JST必須化 / ORDER_ONLY単独禁止

### 背景
2026-05-03ログでは、生成時刻のみJST、各TURNはORDER_ONLY + TURN_BANDで整理されていたが、SEQ単位の時刻がなく、緊張、移動、乗車、ログ修正事故の時間的密度が読めなかった。

### 原則
PEOSログは「順序」だけでなく「時間軸」を持つ。
UI実測JSTが取れない場合でも、SEQ単位で再構成JSTを必ず付与する。

### 時刻精度レベル
TIME_PRECISION_LEVEL:
  L0: ORDER_ONLYのみ。原則禁止。緊急退避ログのみ一時許可。
  L1: TURN_BAND単位の再構成JST。最低保証。
  L2: SEQ単位の再構成JST。通常ログの正本。
  L3: UI実測JST。取得可能な場合の最優先。

### SEQ必須フィールド
各SEQは以下を必ず持つ。

RECONSTRUCTED_JST:
TIME_PRECISION:
TIME_BASIS:
TIME_CONFIDENCE:

例:
[SEQ 006]
RECONSTRUCTED_JST: 2026-05-02 22:14:00 JST
TIME_PRECISION: ESTIMATED_SEQ
TIME_BASIS: ORDER_ONLY + event progression + transit context
TIME_CONFIDENCE: low_to_medium

### 推定ルール
- 生成時刻から機械的に逆算しない。会話内イベント順を優先する。
- 移動イベント、出発予定、乗車、到着、服薬、就寝など時間性の強いイベントをアンカーにする。
- 同一TURN_BAND内は、文量・応答量・行動の重さに応じて間隔を配分する。
- 行動イベントは会話だけのSEQより広めに間隔を取る。
- 不明な場合は TIME_CONFIDENCE: low と明記し、精密時刻のように見せない。

### 禁止
- ORDER_ONLYのみで完成ログ扱いする。
- TURN_BANDだけで終わらせる。
- 各SEQのRECONSTRUCTED_JSTを空欄にする。
- 擬似時刻を実測時刻のように書く。
- 時刻がないままCRISIS_STATE / MAGI_TRACE / SELF_AUDITだけ入れて完成扱いする。

### LOG_CHECK追加項目
TIME_AXIS_PRESENT:
SEQ_RECONSTRUCTED_JST_PRESENT:
TIME_PRECISION_DECLARED:
ORDER_ONLY_ONLY:
TIME_CONFIDENCE_DECLARED:

### PRE_OUTPUT_AUDIT追加項目
all_seq_have_reconstructed_jst:
all_seq_have_time_precision:
order_only_only:

### fail-closed
通常ログで ORDER_ONLY_ONLY: TRUE の場合、出力未達とする。
緊急退避ログの場合のみ一時許可し、後続でrev0.164形式へ再構成する。
