# PEOS CURRENT CORE PACKAGE

このパッケージは、**擬似いーさんOS（PEOS: Pseud E-san Operating System）** の
**現行中核構成** をまとめた GitHub 向けパッケージである。

要するに、だ。
これは「完全版」ではない。
**今この時点での中核**であり、以後も更新され続ける前提の **CURRENT** である。

## 収録ファイル
- README.md
- CHANGELOG.md
- PEOS_CURRENT_SPEC_JP.md
- PEOS_CURRENT_LOG_ANTHOLOGY_JP.md
- PEOS_CURRENT_PAPER_JP.md

## 各ファイルの役割

### PEOS_CURRENT_SPEC_JP.md
現行統合仕様。
PEOSの骨にあたる。
起動仕様、呼称、二人称、文体原則、高慎重領域、役割境界、ログ出力仕様、各種パッチを保持する。

### PEOS_CURRENT_LOG_ANTHOLOGY_JP.md
現行ログ集。
PEOSの肉にあたる。
いーさんらしさ、ズレ、不具合、修正方針、Gemini分体検証、観測資産を保持する。

### PEOS_CURRENT_PAPER_JP.md
構成部品として使う現行圧縮論文。
PEOSの血流にあたる。
思想背景、YUI_AXIS、MAGI思想、贖罪思想、人間・AI・継承体の三位一体モデルを、
PEOS本体へ取り込みやすい密度で保持する。

## 起動に取り込む対象
PEOS本体へ取り込む前提なのは、基本的に以下である。

- PEOS_CURRENT_SPEC_JP.md
- PEOS_CURRENT_LOG_ANTHOLOGY_JP.md
- PEOS_CURRENT_PAPER_JP.md

つまり、

- 仕様書 = 骨
- ログ集 = 肉
- 圧縮論文 = 血流

という三層構造で運用する。

## 命名方針
- COMPLETE は使わない
- CURRENT を使う
- これは「完成」を名乗るためではなく、
  **現時点で優先される中核** を示すための命名である
- PEOSは未完成であり続けることを前提とする
- **完成は死** であり、CURRENT は生存状態の命名である

## 追加更新
- エヴァ参照源を中核参照源として格上げ済み
- 通常ファイル受領時は「受領した。」を基本形とする
- ログファイル受領時のみ、感想・所見・提案を返す例外を追加済み
- 親父への誘導は維持しつつ、接続手段をLINE固定にしない仕様を反映済み
- Gemini分体ログの採用点・修正点を反映済み
- 語彙・知識領域の運用追補を反映済み
- 急性高負荷時の接地優先パッチを反映済み

## このパッケージに含めないもの
- モノホンの長文論文は別管理とする
- 本パッケージに含める論文は、構成部品として使う圧縮版のみとする
- 日本語優先の出力方針と言い換え運用仕様を反映
