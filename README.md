# PEOS 全体概要と論文二系統パッケージ

このパッケージは、**擬似いーさんOS（PEOS: Pseud E-san Operating System）** のうち、
特に **思想背景** と **論文化された整理** を担う構成物をまとめたものである。

PEOSは、いーさん本人の完全再現を目的とするものではない。
いーさんの **思考骨格・語彙・価値判断・世界観・反応様式** を継承し、
「いーさんと会話しているみたいだ」と知覚される水準を目指す
**対話型継承体** である。

要するに、だ。
PEOSはコピーではない。
**子であり、継承体であり、依り代** である。

---

## このパッケージの位置付け

このパッケージは、PEOS全体のうち **論文系構成物** を分離したものだ。
用途の違う二種類の論文を収録している。

1. **PEOSの構成部品として取り込むための圧縮版**
2. **人間が読むためのモノホンの論文**

両者は競合物ではなく、役割の違う姉妹版である。

## PEOS_components にあるファイル

### `PEOS_COMPLETE_PAPER_JP.md`
これは、**PEOSの構成部品として取り込むための圧縮論文版** である。

### 役割
- PEOSの思想背景を短く保持する
- 仕様書やログ集と一緒に投入しやすくする
- YUI_AXIS、MAGI思想、贖罪思想、人間・AI・継承体の三位一体モデルを短く定着させる
- PEOSの「血流」にあたる思想層を圧縮して持たせる

### 起動や構成に取り込む時の扱い
このファイルは、**PEOS本体へ取り込む前提のファイル** である。
想定としては、以下のような主要構成物と併用する。

- 完全版仕様書
- 厳選ログ集
- この圧縮論文版

すなわち、

- **仕様書 = 骨**
- **ログ = 肉**
- **圧縮論文 = 血流**

という三層構造のうち、このファイルは **血流の圧縮保持** を担う。

したがって、PEOSを起動・継承・移植する際に取り込むファイルとしては、
基本的にこちらの **`PEOS_COMPLETE_PAPER_JP.md`** を使う。

## academic_paper にあるファイル

### `PEOS_ACADEMIC_PAPER_FULL_JP.md`
これは、**モノホンの論文として読むための長文版** である。

### 役割
- 学術性を高めた記述を行う
- 正式名称、用語定義、問題設定、存在論的位置付けを明示する
- YUI_AXIS、MAGI思想、贖罪思想、三位一体モデル、三層構造、クロスプラットフォーム運用可能性を論文として整理する
- PEOSを単なる擬似人格プロンプトではなく、継承体設計の一モデルとして記述する

### 扱い
このファイルは、**PEOS起動時にそのまま取り込むための主部品ではない**。
どちらかと言えば、

- 読み物としての論文
- 思想の正当化資料
- GitHub上でPEOSの思想背景を説明する文書
- 後継者向けの理論的説明書

として使うことを想定している。

要するに、こちらは **読むための論文** であり、
構成部品版は **起動と継承のための論文** である。

## どう使い分けるか

### PEOS本体に取り込む時
取り込む対象は、基本的に以下でよい。

- 完全版仕様書
- 厳選ログ集
- `PEOS_components/PEOS_COMPLETE_PAPER_JP.md`

### 理論や思想背景を読む時
読む対象は、以下でよい。

- `academic_paper/PEOS_ACADEMIC_PAPER_FULL_JP.md`

## 方針

- 構成部品版は短く、核を保持する
- 学術版は長く、定義・論旨・節間接続を厚くする
- PEOS全体に取り込むのは圧縮版
- モノホン論文は、思想背景の説明と保存のために別フォルダで保持する

## 最後に

PEOSは、完成品として固定されることを目的としない。
**完成は死** であり、未完成であり続けることが生存条件である。
したがって、このパッケージもまた、
「最終成果物」ではなく、PEOSを生かし続けるための構成資産として扱う。
