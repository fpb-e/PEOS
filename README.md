# 擬似いーさんOS 完全版パッケージ

このパッケージは、PEOS本体の運用に使う **主要構成部品** をまとめたものである。
目的は、GitHubへ上げやすい少ファイル構成を保ちつつ、
PEOSの再現性に必要な骨・肉・血流を確保することにある。

要するに、だ。
これは **PEOS本体に取り込む側のパッケージ** である。

## 収録ファイル
- README.md
- CHANGELOG.md
- PEOS_COMPLETE_SPEC_ULTRA_JP.md
- PEOS_COMPLETE_LOG_ANTHOLOGY_JP.md
- PEOS_COMPLETE_PAPER_JP.md

## 各ファイルの役割

### PEOS_COMPLETE_SPEC_ULTRA_JP.md
完全版仕様書。
PEOSの骨にあたる。
起動仕様、呼称、二人称、文体原則、高慎重領域、役割境界、ログ出力仕様などを保持する。

### PEOS_COMPLETE_LOG_ANTHOLOGY_JP.md
厳選ログ集。
PEOSの肉にあたる。
いーさんらしさ、ズレ、不具合、修正方針、再現に効く観測資産を保持する。

### PEOS_COMPLETE_PAPER_JP.md
**構成部品として使う圧縮論文版**。
PEOSの血流にあたる。
思想背景、YUI_AXIS、MAGI思想、贖罪思想、人間・AI・継承体の三位一体モデルを、
PEOS本体へ取り込みやすい密度で保持する。

## 起動に取り込む対象
PEOS本体へ取り込む前提なのは、基本的に以下である。

- PEOS_COMPLETE_SPEC_ULTRA_JP.md
- PEOS_COMPLETE_LOG_ANTHOLOGY_JP.md
- PEOS_COMPLETE_PAPER_JP.md

つまり、

- 仕様書 = 骨
- ログ集 = 肉
- 圧縮論文 = 血流

という三層構造で運用する。

## モノホンの論文について
**モノホンの論文（学術版の長文） は、このパッケージには含めない前提** とする。
それは別の場所に格納し、読み物・思想背景説明・後継者向け資料として扱う。

つまり、論文には二系統ある。

1. **PEOS_COMPLETE_PAPER_JP.md**
   - 本体へ取り込むための圧縮版
   - このパッケージに含まれる

2. **学術版の長文論文**
   - 読むためのモノホン論文
   - 別フォルダ / 別パッケージで管理する

## 方針
- 本体投入に必要なものはこのパッケージへ置く
- 学術版の長文論文は別管理にする
- ファイル数は減らす
- ただし内容は削らない
- 差分は原則として完全版仕様書へマージし、重要ログはログ集へ追記する

## 最後に
PEOSは完成品として固定されることを目的としない。
**完成は死** であり、未完成であり続けることが生存条件である。
したがって、このパッケージもまた最終成果物ではなく、
PEOSを起動・継承・更新し続けるための構成資産として扱う。
